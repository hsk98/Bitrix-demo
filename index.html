--- layout: default title: Pipelines & Sales Funnels permalink: /pipelines/ ---

Pipelines & Sales Funnels

Residential · Commercial · Leasing

Tailored funnels for different lead types and transaction models. Stage-specific required fields keep data clean and reporting consistent.

Residential Sales

Owner/Buyer representation

  1. New Lead → auto-assign + SLA 30m
  2. Qualification → budget, location, beds, timeline
  3. Property Match → send curated shortlist
  4. Viewing Scheduled
  5. Offer & Negotiation
  6. Agreement/Contract
  7. Won / Lost (reason required)

Residential Leasing

Tenancy cycle

  1. New Inquiry
  2. Qualification & Docs
  3. Viewing
  4. Application & Screening
  5. Offer / Reservation
  6. Contract & Move-in
  7. Won / Lost

Commercial Sales

LOI/MOU heavy with stakeholder mapping

  1. Inbound / Prospect
  2. Qualification (use case, size, power, zoning)
  3. Viewing / Site Visit
  4. LOI / MOU
  5. Legal & Approvals
  6. Contract / Transfer
  7. Won / Lost

Commercial Leasing

Multi-party, often longer cycles

  1. Inbound / Outreach
  2. Qualification (trade license, timeline, fit)
  3. Viewing / Technical
  4. Proposal & Negotiation
  5. HOA / Landlord Approvals
  6. Lease Execution
  7. Won / Lost

Stage-Required Fields (examples)

Qualification

  • Budget / Rent range
  • Bedrooms / Sqft / Power
  • Move-in timeline
  • Lead Type & Source

Viewing

  • Viewing date/time
  • Property ID
  • Agent assigned
  • Attendees

Offer

  • Offer amount / terms
  • Docs received
  • Decision-makers
  • Probability %

Closed

  • Reason Won/Lost
  • Commission
  • Marketing source
  • Cycle time (auto)
--- layout: default title: Automation & SLA Design permalink: /automations/ ---

Robots, triggers, and queues that reduce latency and enforce quality—Bitrix24-ready patterns.

Lead Capture & Routing

  • • Form/portal/webhook → Validate → Enrich (email/phone check)
  • • Dedupe against Contacts/Leads (fuzzy phone/email)
  • • Auto assign by team, source, geo, property class
  • • Start SLA timer: response due in 30m (residential), 1h (commercial)

Nurture & Follow-up

  • • No contact in 24h → create task + notify team lead
  • • Viewing scheduled → send SMS/iCal reminders to client & agent
  • • Offer created → auto-generate quote doc & checklist
  • • Idle 7 days → escalate and recycle to nurture list

Post-Close & Handoff

  • • On Won → tasks for compliance & payout
  • • Closed Lost → capture reason & trigger remarketing
  • • Lease expiry reminders at T-90/T-60/T-30
  • • Client satisfaction pulse (CSAT) 3 days after move-in

Example Bitrix24 Robots & Triggers

  • • Trigger: Incoming call/email → move to Qualification
  • • Robot @ New: Assign by queue; set SLA; send intro message
  • • Robot @ Viewing: Create reminder tasks; send calendar invites
  • • Robot @ Offer: Create quote; notify manager for approval
  • • Robot @ Idle: If no activity 3 days → escalate to team lead
  • • Robot @ Won: Create payout approval + finance notification

--- layout: default title: 90‑Day Plan (Real Estate CRM) permalink: /plan-90/ ---

Goal

Lift lead response speed, pipeline conversion, and data quality while aligning marketing, sales, and operations.

Days 0–30: Assess & Stabilize

  • Audit pipelines, fields, automations, and permissions; document pain points.
  • Define qualification checklist per pipeline; enforce stage-required fields.
  • Implement lead routing & SLA timers; stand up daily SLA report.

Days 31–60: Automate & Enable

  • Deploy nurture sequences and idle/escalation rules.
  • Build dashboards: lead source, funnel, agent productivity, cycle time.
  • Train agents & managers; publish SOPs; create templates (emails, SMS).

Days 61–90: Optimize & Scale

  • Attribution hygiene; UTM standards and campaign reporting.
  • A/B test scoring thresholds; refine routing by performance.
  • Integrate portals/telephony; evaluate new modules; plan Q2 backlog.
